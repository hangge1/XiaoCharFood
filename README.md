# XiaoCharFood

XiaoCharFood 是一个面向情侣和小家庭的微信小程序，用来记录日常饮食、提前规划一日菜单、沉淀去过的餐厅体验。

项目定位不是严格的健康管理工具，而是一个低压力的私人饮食生活助手：帮用户记住最近吃过什么，减少每天“吃什么、做什么、去哪吃”的重复思考。

## 当前版本

当前是本地优先的 MVP 版本，不依赖后端和云开发。数据使用微信小程序本地存储保存，适合先验证核心体验闭环。

主要入口：

- `饭桌`：记录今日饮食，查看最近 7 天低压力回顾。
- `今晚`：根据需求关键词生成完整一日菜单和买菜清单。
- `回味`：记录和搜索去过的餐厅体验。
- `小家`：设置称呼、做饭偏好，查看本地数据状态。

## 功能概览

### 饭桌

- 记录早餐、午餐、晚餐、加餐。
- 支持饮食标签和放纵餐标签。
- 饮食记录包含日期和具体时间。
- 最近 7 天回顾外食、放纵餐、偏油等轻量趋势。
- 首次使用可设置昵称，首页使用昵称问候。

### 今晚

- 输入口味、已有食材、减肥、增肌、高蛋白、少油等需求关键词。
- 内置 40 道常见中餐和轻食菜谱库，当前不依赖 AI。
- 一次性推荐早餐、午餐、晚餐、加餐。
- 自动汇总买菜清单，方便做饭的人提前准备食材。
- 可保存整套一日菜单。
- 可将某一餐推荐直接记录为饮食记录。

### 回味

- 新增餐厅记忆，保存店名、人均、菜品、备注和场景标签。
- 支持从地图选择餐厅地址，并保存经纬度。
- 支持按店名、菜品、备注搜索；输入时展示下拉联想，搜索无结果时可直接记录新餐厅。
- 支持筛选“还想再去”的餐厅。
- 支持获取当前位置，显示附近 5km 内保存过地址的餐厅。
- 附近餐厅会显示在地图标注和列表中，可直接打开地图位置。
- 定位为私人餐厅记忆，不做公共点评社区。

### 小家

- 设置和修改用户称呼。
- 设置做饭口味偏好。
- 查看本地饮食记录、菜单计划、餐厅记忆数量。
- 支持清空本地数据。

## 视觉方向

当前视觉方向是“年轻情侣/小家庭的饭桌生活感”：

- 温暖浅色背景。
- 软卡片和圆角按钮。
- 冰箱贴式 sticker。
- 本地 TabBar 主题图标：饭碗、锅、回味标记、小家。
- 健康相关文案使用“回顾、平衡、建议”等低压力表达，避免惩罚式反馈。

```text
.
├── app.js
├── app.json
├── app.wxss
├── assets/

    └── util.js
```

## 本地运行

1. 使用微信开发者工具打开本项目目录。
2. 导入 `project.config.json`。
3. 在模拟器中编译运行。

当前项目没有配置 `package.json` 或 `miniprogram-ci`，因此暂不支持命令行构建。

## 验证方式

已使用 Node 做基础语法检查：

```powershell
node --check app.js
node --check utils/store.js
node --check utils/recommender.js
node --check pages/index/index.js
node --check pages/cook/cook.js
node --check pages/restaurants/restaurants.js
node --check pages/mine/mine.js
```

推荐逻辑和本地数据层也通过 mock `wx` 做过 smoke test。

## 已知限制

- 数据仅保存在本机，没有云同步。
- 暂不支持家庭/情侣共享编辑。
- 暂不支持图片上传存储。
- 地图能力依赖微信小程序位置授权；未保存地址的餐厅不会出现在附近地图中。
- 一日菜单推荐是固定菜谱库 + 本地规则推荐，不是 AI 或专业营养建议。
- `减肥`、`增肌` 等关键词仅作为生活化饮食偏好使用，不提供医疗或专业营养诊断。
